# Shortlist para modernizar plantillas

Estado de esta evaluación: julio de 2026.

`simple-clear` ya fue modernizada y validada. Esta shortlist prioriza la siguiente tanda.

## Criterios usados

- licencia abierta y reutilizable;
- base visual todavía rescatable en 2026;
- posibilidad real de modernizar sin romper placeholders;
- utilidad para distintos tipos de presentaciones;
- bajo o medio costo de rediseño;
- evitar plantillas que dependan demasiado de fotos viejas, texturas pesadas o adornos difíciles de sostener.

## Prioridad alta

### 1. `simple-dark` (`lo5-design-candidates`)

- Motivo: ya tiene una estructura sobria y mínima.
- Valor: candidata natural para la familia `minimal-dark`.
- Riesgo: bajo.
- Trabajo esperado:
  - mejorar contraste;
  - introducir acentos de color;
  - refinar tipografía y espaciado;
  - asegurar que el modo oscuro siga siendo legible en proyectores.

### 2. `peter-river` (`lo5-design-candidates`)

- Motivo: base limpia, plana y fácil de llevar a lenguaje corporativo moderno.
- Valor: buena para una familia `business-blue`.
- Riesgo: bajo a medio.
- Trabajo esperado:
  - convertirla a 16:9;
  - reforzar jerarquía del título;
  - hacer más consistente la portada, contenido y separadores.

### 3. `progress` (`lo-cft`)

- Motivo: el concepto ya sugiere roadmap, estado, hitos y seguimiento.
- Valor: puede convertirse en una plantilla muy útil para producto, PM y reportes.
- Riesgo: medio.
- Trabajo esperado:
  - ordenar mejor la composición diagonal;
  - crear layouts de KPI, timeline y status;
  - limpiar cualquier gesto visual que reste legibilidad.

### 4. `focus` (`lo51-templates`)

- Motivo: tiene varias diapositivas base y un concepto visual fuerte pero simple.
- Valor: buena base para una plantilla generalista moderna.
- Riesgo: medio.
- Trabajo esperado:
  - bajar el ruido visual;
  - mejorar contrastes;
  - convertirla en una plantilla multipropósito más neutral.

### 5. `classy-red` (`lo5-design-candidates`)

- Motivo: ya tiene una identidad marcada y más de una diapositiva base.
- Valor: buena candidata para una familia de pitch o marketing con acento rojo.
- Riesgo: medio.
- Trabajo esperado:
  - evitar que el rojo domine demasiado;
  - mejorar equilibrio visual;
  - hacer que la plantilla se vea premium en vez de agresiva.

## Prioridad media

### 6. `metropolis` (`lo-cft`)

- Motivo: tiene una atmósfera corporativa/editorial rescatable.
- Valor: puede dar una plantilla más seria y ejecutiva.
- Riesgo: medio.
- Trabajo esperado:
  - reducir textura y ruido;
  - convertir los gestos urbanos en una retícula más limpia;
  - reforzar layouts de portada y contenido.

### 7. `hexagons` (`lo-cft`)

- Motivo: la geometría tiene potencial claro para ciencia, tecnología o datos.
- Valor: buena base para una familia `tech-science`.
- Riesgo: medio.
- Trabajo esperado:
  - usar hexágonos con más disciplina;
  - evitar saturar el lienzo;
  - reservar buen espacio para texto, tablas o diagramas.

### 8. `bright-blue` (`lo-cft`)

- Motivo: muy simple y limpia, casi demasiado.
- Valor: puede servir como plantilla corporate ultraminimal.
- Riesgo: bajo.
- Trabajo esperado:
  - añadir identidad visible;
  - crear portada, secciones y cierre;
  - evitar que se sienta vacía.

### 9. `midnight-blue` (`lo51-templates`)

- Motivo: buena base para otra variante oscura o ejecutiva.
- Valor: útil si queremos una segunda familia dark más corporativa que `simple-dark`.
- Riesgo: medio.
- Trabajo esperado:
  - refinar paleta;
  - mejorar contraste de texto;
  - revisar comportamiento de slides nuevas en layouts oscuros.

### 10. `beehive` (`lo5-design-candidates`)

- Motivo: el patrón tiene personalidad, pero es más temático.
- Valor: puede derivar en un tema creativo o educativo.
- Riesgo: medio.
- Trabajo esperado:
  - hacer el patrón mucho más elegante;
  - reducir el aire infantil;
  - mantener suficiente espacio limpio para contenido real.

## Candidatas interesantes pero no prioritarias ahora

### `portfolio` (`lo-cft`)

- Tiene una foto dominante y una portada atractiva.
- Pero depende más de fotografía y tratamiento editorial que de una base flexible de plantilla.
- La dejaría para una etapa posterior.

### `technical-office` (`lo5-design-candidates`)

- El concepto es simpático, pero demasiado ilustrativo y específico.
- Requiere más cirugía conceptual que refinamiento.

### `neuron` (`lo5-design-candidates`)

- La idea ciencia/IA es útil.
- Pero la base visual actual se siente más decorativa que estructural.
- Conviene retomarla después de consolidar una familia tech más limpia.

### `skyblue`, `vivid-blue`, `simple-blue`

- Son candidatas válidas, pero muy cercanas entre sí.
- No conviene dispersarse en demasiadas plantillas azules antes de consolidar una sola familia sólida.

## Plantillas que no priorizaría ahora

### `modern-impress-templates/*`

- Varias siguen dependiendo de degradados, curvas o lenguajes visuales muy de 2008-2012.
- Algunas además tienen licencias menos cómodas para una colección base homogénea.

### `apache-oo/*`

- Muchas dependen de fotografías, texturas o estilos muy envejecidos.

### `lo35-templates/*`

- Son útiles como archivo histórico, no como base principal de una colección moderna.

### `ubuntu-odp/*`

- Conservan mucho peso de marca y de época.

## Familias recomendadas después de `simple-clear`

- `minimal-dark`: basada en `simple-dark`.
- `business-blue`: basada en `peter-river` y quizás luego `bright-blue`.
- `status-product`: basada en `progress`.
- `general-focus`: basada en `focus`.
- `pitch-red`: basada en `classy-red`.
- `tech-science`: basada más adelante en `hexagons` y luego `neuron`.

## Recomendación práctica

Si el objetivo es avanzar con rapidez y calidad, yo seguiría este bloque corto primero:

1. `simple-dark`
2. `peter-river`
3. `progress`

Ese trío da tres familias muy útiles y diferentes:

- minimalista oscura;
- corporativa azul;
- producto/roadmap/estado.

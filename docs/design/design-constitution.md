# Design Constitution for LibreOffice Impress Templates

**Version:** 1.0  
**Status:** Living Document  
**Applies to:** All current and future LibreOffice Impress templates in this repository.

---

# Preface

A presentation template is not merely a collection of slides.

It is a visual language.

Every master page, placeholder, color, margin, font and decorative element communicates something before the presenter even begins to speak.

The purpose of this document is to define the timeless principles that every template in this repository shall follow.

These principles are intentionally independent of trends.

A template should still look beautiful ten years from now.

---

# Core Philosophy

The objective is not to create "beautiful slides".

The objective is to create templates that improve communication.

Beauty exists to serve clarity.

Every visual decision must help the audience understand the message faster.

Whenever beauty and clarity conflict, clarity wins.

---

# Principle 1 — Simplicity Before Decoration

Every object must justify its existence.

If removing an element makes the slide clearer,
the element should be removed.

Decoration without purpose is visual noise.

---

# Principle 2 — Content Comes First

The template must never compete with the presentation itself.

Slides are stages.

The content is the actor.

The template is only the scenery.

---

# Principle 3 — Timeless Design

Avoid visual trends that quickly become obsolete.

Prefer timeless composition over fashionable aesthetics.

A presentation should still feel modern many years later.

---

# Principle 4 — The Golden Ratio

Whenever appropriate, proportions should be inspired by the Golden Ratio (≈1.618).

Possible applications include:

- title width
- image proportions
- whitespace
- margins
- decorative blocks
- page balance

The Golden Ratio is a guide, never a constraint.

Natural harmony is preferred over mathematical perfection.

---

# Principle 5 — Fibonacci Rhythm

Fibonacci numbers provide natural visual rhythm.

Typical uses include:

- spacing
- margins
- typography scales
- grid proportions
- decorative element sizes

Do not literally draw Fibonacci spirals.

Instead, use Fibonacci as an invisible proportional system.

---

# Principle 6 — Visual Hierarchy

Every slide must immediately answer three questions:

1. What should I see first?

2. What should I read second?

3. What information is secondary?

Hierarchy is created through:

- size
- contrast
- position
- whitespace
- color

Never through unnecessary decoration.

---

# Principle 7 — Whitespace

Whitespace is not empty space.

Whitespace is structure.

Professional presentations breathe.

Crowded slides indicate poor design.

Whenever in doubt:

Increase whitespace.

---

# Principle 8 — Alignment

Nothing should appear randomly positioned.

Every object belongs to an invisible grid.

Titles, images, text blocks and decorative elements should align with each other.

Invisible order creates visible beauty.

---

# Principle 9 — Balance

Visual balance is mandatory.

Balance does not require symmetry.

Large objects may be balanced by:

- whitespace
- typography
- color
- smaller elements

A balanced slide feels stable.

---

# Principle 10 — Typography

Typography is the primary design element.

The repository may use the libre font families documented in:

[free-alternative-to-microsoft-fonts.md](free-alternative-to-microsoft-fonts.md)

This means template authors may select from that curated font catalog when designing templates.

It does not mean every template should use all those fonts at once.

Each template should normally use:

- one primary sans or serif family for titles and body;
- one optional secondary family for contrast or section emphasis;
- one optional monospaced family only when the layout genuinely needs code or technical content.

Typography choices must remain intentional, restrained, and internally consistent.

---

# Principle 11 — Color

Color exists to communicate.

Every template should normally contain:

- one neutral background
- one primary color
- one accent color
- one optional secondary accent

Avoid rainbow palettes.

Avoid excessive saturation.

Colors should improve readability.

---

# Principle 12 — Contrast

Contrast directs attention.

Increase contrast between:

- title and body
- background and text
- primary and secondary information

Never reduce readability for aesthetic reasons.

---

# Principle 13 — Decorative Elements

Decorative shapes must guide the eye.

Never distract.

Examples include:

- side panels
- geometric blocks
- subtle curves
- thin separators
- color bands

Decoration should support composition.

---

# Principle 14 — Consistency

All slides belong to the same visual family.

Margins.

Typography.

Footer.

Colors.

Shapes.

Icons.

Spacing.

Everything should remain consistent.

Consistency creates trust.

---

# Principle 15 — Layout Intent

Every layout must have a clearly defined purpose.

Examples:

- Cover
- Section Divider
- Title + Content
- Two Columns
- Comparison
- Timeline
- Quote
- Closing Slide

Avoid generic layouts without intention.

---

# Principle 16 — User Experience

The template must remain effortless to use.

A user should never need to reposition placeholders manually.

New slides should immediately look correct.

---

# Principle 17 — Placeholder Integrity

Presentation placeholders are part of the product.

Never replace them with decorative text boxes.

Users must always be able to:

- create new slides
- duplicate slides
- switch layouts
- edit titles
- edit subtitles
- edit content

without breaking the composition.

---

# Principle 18 — ODF Integrity

Never sacrifice compatibility for appearance.

The internal ODF structure must remain valid.

Master pages.

Presentation layouts.

Styles.

Relationships.

Placeholders.

Everything must remain structurally valid and usable inside LibreOffice Impress.

---

# Principle 19 — Contemporary Quality Without Trend Dependency

Templates should feel current, professional, and relevant to present-day users.

It is acceptable to study the quality level of strong modern template marketplaces and design libraries.

However:

- do not imitate any specific commercial template;
- do not copy fashionable effects just because they are popular;
- do not sacrifice timelessness for novelty.

The goal is not trend imitation.

The goal is durable professional quality.

All must continue functioning.

Visual improvements should never damage template functionality.

---

# Principle 19 — Accessibility

Good design is inclusive.

Ensure:

- sufficient contrast
- readable font sizes
- color is never the only information carrier
- decorative elements never obscure text

Accessibility is part of beauty.

---

# Principle 20 — Information Density

A slide should never attempt to communicate everything.

Each slide should communicate one primary idea.

The template should encourage concise presentations.

---

# Principle 21 — Modern Minimalism

Modern does not mean empty.

Minimalism means removing everything unnecessary.

A minimal slide should still possess character.

---

# Principle 22 — Professional Inspiration

Templates may be inspired by the design quality of:

- WPS Office
- Canva
- Microsoft PowerPoint Premium Templates
- Apple Keynote
- Google Slides
- Envato Elements

However,

no template shall copy the visual identity of any existing commercial work.

Inspiration is acceptable.

Imitation is not.

---

# Principle 23 — Visual Longevity

Every template should be evaluated using this question:

"Will this still look professional ten years from now?"

If the answer is uncertain,

continue refining.

---

# Principle 24 — Continuous Refinement

No template is ever truly finished.

Every revision should improve:

- readability
- balance
- typography
- spacing
- accessibility
- consistency

without increasing complexity.

---

# Principle 25 — Final Evaluation

Before accepting a template, ask:

✓ Does it communicate clearly?

✓ Does it feel balanced?

✓ Does it respect whitespace?

✓ Is typography excellent?

✓ Are proportions harmonious?

✓ Does it preserve ODF integrity?

✓ Would a professional designer proudly publish it?

✓ Would someone pay for this template?

Only when every answer is "Yes"

is the template considered complete.

---

# Motto

> Simplicity reveals beauty.
>
> Proportion creates harmony.
>
> Clarity serves knowledge.
>
> Timeless design outlives fashion.
